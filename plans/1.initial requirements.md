# AUDCAD FOR THE WIN - Initial Requirements

## 1. Project Overview

Build a **standalone MT5 Expert Advisor** that trades **AUDCAD only** on a **cent account**, replicating the grid-recovery basket pattern previously consumed via copy-trade from master account `#50000005`. The goal is to remove the dependency on the master account by reproducing the strategy's behavior in our own EA, while enforcing a hard **20% equity drawdown cap** that the original copy stream did not have (see the 1/31/2026 "order compression" event in the source data, equivalent to ~ -$433.66 real on the cent account).

The first entry signal - what triggers the probe - is intentionally left **TBD** in this document. The source CSV is a copy stream and contains no signal information; the trigger will be selected in the next planning phase.

## 2. Account Type & Symbol Convention

- Default target: **cent account**, symbol **`AUDCAD.c`**. The `.c` suffix is the broker's cent-symbol marker (commonly used by FBS / RoboForex / Exness-style brokers).
- Symbol suffix is configurable via input parameter so the EA can attach to brokers with different naming.
- Code MUST read `SymbolInfoDouble(symbol, SYMBOL_TRADE_CONTRACT_SIZE)` at runtime; cent symbols typically expose 1,000-unit contracts (vs 100,000 for standard) and any pip-value math must use the live contract size.
- All default lot inputs in this document are in **cent lots** (1 cent lot = 0.01 standard lot, 1,000 base-currency units per cent lot at the cent symbol). Logs must show both cent-lot and standard-lot equivalents to prevent the misreading we hit during requirements analysis.
- A standard-account profile (defaults divided by 100) is **out of scope for v1**, planned for v1.x.

> **v1.3 status (2026-05-15)**: v1.0–v1.2 deviated to the standard symbol `AUDCAD#` for the master-account reverse-engineering phase (tuned for ~$50k). **v1.3 returns to the cent default specified above** so the strategy runs from $1,000 USD of real capital. Standard remains supported via the `ProbeLot > 0` override input. See [../strategy/AUDCAD_M15_v1.3.md](../strategy/AUDCAD_M15_v1.3.md) §3.

## 3. Platform & Environment

- **MetaTrader 5** only. No MT4 build.
- **Hedging-mode** account (required so independent long and short baskets can coexist).
- Pure **MQL5** source, no external DLLs.
- Broker requirement: supports `AUDCAD.c` cent symbol with hedging and acceptable swap rates.

## 4. Symbol Scope

- **AUDCAD only** in v1. No multi-symbol logic, no symbol-list inputs.

## 5. Strategy Summary

A grid-recovery basket strategy:

1. Probe: a small first entry in one direction.
2. If price moves against the probe by a fixed pip step, add a much larger position in the same direction.
3. Continue adding at fixed pip intervals, with each successive add larger than the last by a linear increment (not a x2 martingale).
4. Close **all** positions in the basket together when the basket's weighted-average entry has recovered into a small profit (or when the emergency-exit rule fires).
5. Long and short baskets are independent and may exist simultaneously.

Per direction, only one open basket at a time.

## 8. Exit Rules

- **Basket TP**: close all legs in the basket when, for a sell basket, `weighted_avg_entry - bid >= TP_buffer_pips`; for a buy basket, `ask - weighted_avg_entry >= TP_buffer_pips`. `TP_buffer_pips` is configurable, default **5 pips**, intended to cover commission + accumulated swap with a small profit.
- **No per-leg TP / per-leg SL** in v1 - the basket is the unit of risk and the unit of exit.
- **Optional time-stop**: close the basket if held longer than `MaxBasketHoldHours` regardless of price. Default **off**, but the timer is always recorded so its impact can be evaluated. Motivated by observed cent-account swap drag (e.g., -$2.71 real on a single 0.108 std-lot leg over 36 h).

## 9. Risk Management - 20% Equity Drawdown Cap

This is the v1 hard guardrail and overrides every other rule.

- On every tick:
  - Compute `basket_floating_loss = sum_open_positions(P/L_basket_currency)`.
  - Compute `dd_fraction = -basket_floating_loss / account_equity` (positive when in loss).
- **Block-add rule**: if executing the next ladder step would push `dd_fraction` above 20% at *current* price, do not place the add. Mark the basket as "ladder exhausted".
- **Emergency exit**: if `dd_fraction >= 20%`, close the entire basket at market immediately. This is the line that prevents a repeat of the 1/31/2026 order-compression event.
- **Pre-trade sizing**: at probe time, the EA must verify that *probe + full ladder, evaluated at the price where the last add would fire, plus an N-pip safety pad* remains within the 20% cap. If the ladder cannot fit, the probe size auto-reduces in cent-lot steps until it does (or down to `SYMBOL_VOLUME_MIN`, after which the probe is skipped and an event logged).
- Hard floor: probe `>= SYMBOL_VOLUME_MIN` of the cent symbol (typically 0.01 cent lot).

> **Realised in v1.3** via `ComputeBaseLot()` — a closed-form base lot such that the full 10-leg ladder's worst-case loss equals exactly `MaxDDPct × equity` at the FitPad-extended worst price. Uses `MathFloor` (not `MathRound`) to guarantee the result never exceeds the cap by one `vol_step`. See [../strategy/AUDCAD_M15_v1.3.md](../strategy/AUDCAD_M15_v1.3.md) §2 for the derivation; the `WC = 44,627` lot-pip constant is recomputed at every `OnInit` from `MaxLegs`, `GridStepPips`, `FitPadPips`, and the locked multipliers.

## 10. Account / Money Management

- **Auto-scale probe size from cent-account equity**: `probe_cent_lots = max(SYMBOL_VOLUME_MIN, floor(equity / ScaleDivisor) x SYMBOL_VOLUME_STEP)`. `ScaleDivisor` is a configurable input.
- The same `scale` factor is used to scale the linear-additive increment in Section 7 so the whole ladder grows/shrinks coherently with equity.

> **v1.3 supersedes the `ScaleDivisor` heuristic** with a closed-form replacement that targets the §9 DD budget exactly:
> ```
> base = floor( (equity × MaxDDPct/100) / (WC × PipValPerLot()), vol_step )
> ```
> where `WC = Σ m(n) × adv_pips(n) = 44,627` lot-pips per unit base lot for the locked 12×N / 10-leg / 22-pip-step ladder. The constant is recomputed at OnInit from the live inputs so future tweaks to `MaxLegs`, `GridStepPips`, `FitPadPips`, or the multipliers re-derive it. Worked $1k cent example: `base = 0.06`, projected ladder `[0.06, 1.44, 2.16, …, 7.20]`, `wc_pct ≈ 18.7%`. Full derivation: [../strategy/AUDCAD_M15_v1.3.md](../strategy/AUDCAD_M15_v1.3.md) §2.
- **Swap-awareness**: the EA tracks running swap per basket and emits a warning event when projected swap exceeds 1% of equity. (Cent accounts are disproportionately sensitive to swap because swap is paid at full broker scale relative to small position notional.)

## 11. State & Persistence

- Basket state must survive EA restart: positions live on the broker, and the EA re-attaches by **magic number per direction** plus comment tag.
- Magic numbers: `LongMagic` and `ShortMagic` (configurable; defaults `50000051` and `50000052`). **Do not** reuse the master account number `50000005` as the magic - this EA is the producer, not a consumer of someone else's signal.
- On startup, EA scans existing positions matching its magics, reconstructs the basket state (weighted-average entry, ladder index, equity peak), and resumes management without placing new orders until a tick arrives.

## 12. Logging & Telemetry

- Log every probe, add, basket close, emergency exit, and block-add event with: timestamp, direction, price, cent lots, std-lot equivalent, account equity, equity %, basket weighted-average breakeven, ladder index, swap accumulated.
- Output a CSV log structurally compatible with the source `AUDCAD_History_*.csv` so post-hoc analysis can be repeated with the same tooling.
- Include a column `account_type=cent|standard` in the log header so future analysis cannot misread cent values as standard. (This is the exact ambiguity that nearly derailed requirements analysis.)

## 15. Acceptance Criteria for v1

1. The EA, attached to a fresh cent demo account on `AUDCAD.c`, opens, manages, and closes baskets whose **shape** (probe -> linear-additive ladder -> joint close at weighted-average breakeven + buffer) matches the patterns in the source CSV - without any external copy-trade input.
2. On synthetic adverse-move stress runs, **equity drawdown never exceeds 20%** of starting equity, including the worst observed historical move.
3. All ladder, spacing, exit, and risk parameters are exposed as configurable inputs with defaults matching this document.
4. Logs are emitted in cent-lot units with standard-lot equivalents and the `account_type` column.
5. Restart safety: stopping the EA mid-basket and re-attaching reconstructs basket state and resumes management without duplicate orders.
