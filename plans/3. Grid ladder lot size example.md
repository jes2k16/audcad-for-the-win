## 1. The two modes — side-by-side

| | **Auto = true** (v1.4 default) | **Auto = false** |
|---|---|---|
| `base` source | Recomputed at probe-open from equity, `Max_Drawdown_Percentage`, broker pip value | Read verbatim from `Default_Base_Lot_Size` |
| Scales with equity? | Yes — bigger account ⇒ bigger base | No — fixed forever |
| WC loss as % of equity | Bounded by `Max_Drawdown_Percentage` (e.g. 35%) | Drifts: dangerous on a too-small account, tiny on a too-big one |
| Skips when too small? | Yes — logs `SKIP_PROBE / base_below_vol_min` | No — opens anyway |
| Cached on basket | Yes — every grid add uses the cached base | Yes — same input feeds every add |
| Typical use | Canonical XM cent deployment | v1.2 standard regression, broker-unit pinning, manual control |

> Both modes share the **same** multiplier shape and the **same** `LegLot(n, base)` formula — only `base` differs.

## 3. When to use which mode

| Goal | Setting |
|---|---|
| Canonical XM cent deployment | **Auto = true** |
| Replay v1.2 standard-account backtest (G13) | Auto = false, `Default_Base_Lot_Size = 0.01`, `Abort_If_Standard_Account = false`, `Max_Drawdown_Percentage = 20`, symbol `AUDCAD#` |
| Pin a known base while sanity-checking a new broker | Auto = false, `Default_Base_Lot_Size = 0.10` |
| Unusual broker where `[UNIT_SANITY]` shows a non-standard cent denomination | Auto = false, hand-picked `Default_Base_Lot_Size` |

## 4. Buy / Sell signal — what fires a probe

All conditions evaluated on the **last completed M15 bar** (shift = 1, close-level values only — no intra-bar peeking).

### 4.1 BUY condition

```
RSI(14) < 50                         ← Buy_Only_If_RSI_LessThan
AND any of:
    StochRSI %K ≤ 20                 ← Buy_Fire_If_StochRSI_K_LessThan_EqualTo
    OR Bollinger %B ≤ 0.10           ← Buy_Fire_If_Bollinger_LessThan_EqualTo
    OR RSI(14) ≤ 40                  ← Buy_Fire_If_RSI_LessThan_EqualTo
```

Reading: bias must be on the bearish/oversold side of RSI 50 (the direction gate), **and** at least one of the three "deep" conditions must confirm an oversold extreme.

### 4.2 SELL condition

```
RSI(14) > 50                         ← Sell_Only_If_RSI_GreaterThan
AND any of:
    StochRSI %K ≥ 60                 ← Sell_Fire_If_StochRSI_K_GreaterThan_EqualTo
    OR Bollinger %B ≥ 0.90           ← Sell_Fire_If_Bollinger_GreaterThan_EqualTo
    OR RSI(14) ≥ 60                  ← Sell_Fire_If_RSI_GreaterThan_EqualTo
    OR distance to 500-bar swing high ≤ 50 pips   ← Swing_High_DistPips
```

Reading: bias must be on the bullish/overbought side of RSI 50, **and** at least one of the four "deep" conditions confirms — three indicator extremes plus a structural "price is near a rolling swing high" check that catches breakouts the indicators alone miss.

> BUY and SELL are mutually exclusive on any given bar — `RSI < 50` and `RSI > 50` cannot both hold.

### 4.3 Indicator parameters (defaults — v1.4 input names)

| Indicator | Parameter | v1.4 input | Default |
|---|---|---|---:|
| RSI | period (Wilder) | `RSI_Period` | 14 |
| Bollinger Bands | period | `Bollinger_Period` | 20 |
| Bollinger Bands | std-dev | `Bollinger_StdDev` | 2.0 |
| StochRSI | RSI lookback (= stoch window) | `Stoch_RSI_Lookback` | 14 |
| StochRSI | %K smoothing | `Stoch_RSI_Smooth` | 3 |
| Rolling swing high | bars looked back | `Swing_High_Bars` | 500 |
| Rolling swing high | "near" distance | `Swing_High_DistPips` | 50 pips |

### 4.4 HTF gate (v1.1 addition — applies to **new probe (level 1 only)**)

A probe is allowed only if the higher-timeframe trend agrees with the signal direction:

| Side | Gate condition |
|---|---|
| BUY  | D1 close > D1 EMA(20), shift = 1, strict `>` |
| SELL | D1 close < D1 EMA(20), shift = 1, strict `<` |

Gate failure logs `GATE_BLOCK` and the signal is skipped. The gate is **not** consulted when adding to a live basket or when checking the exit — only at the moment a brand-new probe is about to open. Toggle with `Enable_HTF_Gate` (default `true`).

### 4.5 What happens when a signal fires (v1.2 single-basket rules)

```
On every M15 bar close, after signal evaluation:

IF a basket IS open:
    ignore all signals — basket runs until its weighted-avg + 10 pip target hits
    (or the emergency-DD market close fires)

IF NO basket open AND BUY signal fired AND HTF gate passes (D1 > EMA20):
    → ComputeBaseLot (auto) OR read Default_Base_Lot_Size (override)
    → FitCheck the ladder against current equity & MaxDD cap
    → open BUY probe (level 1 from §2 table)

IF NO basket open AND SELL signal fired AND HTF gate passes (D1 < EMA20):
    → same as above, opposite direction

Signal fires but gate fails       → log GATE_BLOCK, skip
Probe sized below vol_min         → log SKIP_PROBE / base_below_vol_min, skip
```

Bidirectional trading (v1's "BUY basket + SELL basket simultaneous") was removed in v1.2 — only one basket exists at a time.

### 4.6 Why this rule — basis

| Source | Why we trust it |
|---|---|
| Master account #50000005 (130 probes, Feb–May 2026) reverse-engineered to this exact ruleset | The rule was not designed — it was **discovered** to match what the source EA actually did |
| G1 in-sample (Feb 2026, 44 probes) | **91 % hit-rate** — above the 85 % gate |
| G2 out-of-sample (Mar–May 2026, 84 probes) | **85 % hit-rate** — only −6.4 pp drift from in-sample, above the 80 % gate |
| Symmetry of BUY vs SELL | Three indicator extremes per side, with a structural swing-high check added to SELL to catch breakout exhaustion the indicators alone miss (15 pp better recall in trending regimes) |
| Why a direction gate + a "deep" trigger | Either alone over-fires. The direction gate (`RSI ⋚ 50`) ensures we're on the right side of the median; the deep trigger ensures we've actually reached a mean-reversion zone, not just crossed 50 |
| Why D1 EMA20 (v1.1) | Filters mean-reversion shorts against a strong D1 uptrend (and vice-versa) — the regime context single-bar indicators don't see |
| Why +10 pip net exit (v1.2) | Locks the recovery target instead of waiting for an opposite signal that may never come; absorbs the asymmetry between "open" signals and "close" signals revealed by G6 (59–73 % cross-direction match — strong but not bankable) |

Open questions on the signal itself live in [strategy/AUDCAD_M15_v1.md §6](../strategy/AUDCAD_M15_v1.md) (regime-aware buy thresholds, the unexplained ~10–15 % miss rate, grid outliers).
